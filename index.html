Overthrow

A tiny, no-frills, framework-independent, targeted overflow: auto polyfill for use in responsive design.

What is this all about?

You want to use CSS overflow in your designs, but overflow doesn't work properly in many browsers, particularly mobile ones. Many popular mobile browsers treat overflow: auto the same as overflow: hidden, cropping overflow content from view, and leaving users no way to access it.

However, many browsers support overflow very well! In particular, desktop browsers tend to support it without a hitch, and many of the latest versions of most mobile platforms do too. The trouble is, it's hard – perhaps impossible – to test for overflow support, and even if we could use it safely where supported, many popular browsers would be left with a less slick experience (Android 2.1-2.3, BlackBerry 6, and iOS4, to name a few).

What is Overthrow?

Overthrow.js is a conservative, forward-leaning attempt at filling the gaps in overflow support, with an emphasis on letting native implementations do their thing. Out of necessity, Overthrow examines the user agent string to whitelist platforms that have overflow native support, but not before checking through more reliable means: namely, iOS5's touch scrolling CSS property, and a broad desktop browser inference test (no touch support with a screen greater than 1200px wide).

First and foremost, Overthrow is designed to simply enable native CSS overflow in browsers it deems to have native support, but it includes a touch-scrolling polyfill too! In platforms that don't appear to support overflow natively, and support touch events, Overflow will polyfill the feature, allowing smooth, momentum-based scrolling in browsers that'd otherwise be left with an innaccessible experience.

Perhaps most importantly, Overthrow enables overflow support as a qualified, progressive enhancement, meaning your content starts out tall and accessible, and properties like width and height are assigned only in browsers where overflow content can be scrolled (either natively or polyfilled). In browsers that don't support overflow natively, and also don't support touch events, the content will not receive dimensions at all.

Features

  • Lightweight JavaScript: 1kb after min/gzip
  • Dual MIT / GPLv2 license: use it wherever you want.
  • Safe for use in cross-device development
  • Framework-independent: use with any JS library, or none at all!
  • Rather than simulating scrolling with CSS properties, the native JavaScript scrollTop and scrollLeft properties are used. This means any you can use any standard web conventions for scrolling an overthrow element, such as #fragment anchor links, and so on.
  • Form elements are safe: Overthrow doesn't interfere with form controls
  • Embeddable: just like standard CSS overflow, you can embed overthrow elements inside one another.

How do I use it?

Get the Code on Github 1kb

First, download and reference overthrow.js from your document. Anywhere's fine.


<script src="overthrow.js"></script>

Then put a class of overthrow on any elements in which you'd like to apply overflow scrolling.


<div id="foo" class="overthrow">Content goes here!</div>

In browsers that Overthrow.js deems capable of scrolling overflowed content (either natively, or using its polyfill), it will add a class of overflow to the HTML element. Add this CSS to your stylesheet somewhere, enabling overflow on all elements in your document that have an overflow class.


.overthrow .overthrow {
	overflow: auto;
	-webkit-overflow-scrolling: touch;
}

Lastly, design away! Any time you want to set dimensions on an element to trigger overflow scrolling, just be sure to key off that overflow class on the HTML element.


.overthrow #foo {
	height: 280px;
}

API

Overflow adds one variable to the global namespace: overthrow. It has the following methods and properties:

  • overthrow.set: function to initialize overthrow behavior and style in the page. Adds "overthrow" class to HTML element, and touch listener for polyfilled browsers
  • overthrow.forget: function to destroy overthrow behavior and style in the page. Removes "overthrow" class from HTML element, and touch listener for polyfilled browsers
  • overthrow.support: string representing the browser's overthrow support type: "native", "polyfilled", or "none"

That's about it.
